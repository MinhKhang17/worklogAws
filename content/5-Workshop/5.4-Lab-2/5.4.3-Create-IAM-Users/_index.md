## Step-by-Step Guide: Create IAM Users

### Objective
Create individual IAM users for testing the marketplace with different roles: one data provider user and one data consumer user.

---

## Step 1: Open IAM Users Console

1. In AWS IAM Console, on left menu, click **Users**
2. You'll see the users dashboard
3. Click **Create user** button

![](/worklogAws/images/5-Workshops/lab2/task3/step1.png)

---

## Step 2: Create Data Provider Test User

### Step 2.1: Configure User Details

1. In **User name**, enter: `provider-test-user`
2. Check ✓ **Provide user access to the AWS Management Console**
3. Select **I want to create an IAM user** (recommended)
4. In **Console password**, select **Auto-generated**
5. Uncheck ✓ **Users must create a new password at next sign-in** (for testing)
6. Click **Next** button

![](/worklogAws/images/5-Workshops/lab2/task3/step2.1.png)

---

### Step 2.2: Assign Provider Role

1. You're on **Set permissions** page
2. Select **Add user to group** (if groups exist) or **Attach policies directly**
3. In the search box, search for: `EvDataProviderRole`
4. You should see the role we created - but wait, we need to assign the role through groups for users
5. Let's use **Attach policies directly** instead
6. Search for: `EvDataProviderPolicy`
7. Check ✓ the policy
8. Click **Next** button

![](/worklogAws/images/5-Workshops/lab2/task3/step2.2.png)

---

### Step 2.3: Review and Create Provider User

1. You're on **Review and create** page
2. Review all settings
3. Click **Create user** button


---

### Step 2.4: Save Provider User Credentials

1. You'll see success page with credentials:
   - User name: `provider-test-user`
   - Console sign-in URL
   - Password (autogenerated)

2. **Important:** Save these credentials securely (you can display credentials or send email)
3. Make note of the sign-in URL for later testing


---

## Step 3: Create Data Consumer Test User

### Step 3.1: Create Consumer User

1. Click **Create user** button
2. In **User name**, enter: `consumer-test-user`
3. Check ✓ **Provide user access to the AWS Management Console**
4. Select **I want to create an IAM user**
5. Select **Auto-generated** password
6. Uncheck ✓ **Users must create a new password at next sign-in**
7. Click **Next**


---

### Step 3.2: Assign Consumer Policy

1. On **Set permissions** page
2. Select **Attach policies directly**
3. Search for: `EvDataConsumerPolicy`
4. Check ✓ the policy
5. Click **Next**


---

### Step 3.3: Create Consumer User

1. Review settings
2. Click **Create user**


---

### Step 3.4: Save Consumer Credentials

1. Save the credentials for `consumer-test-user`
2. Note the sign-in URL


---

## Step 4: Create Access Keys for Programmatic Access

We need to create access keys for these users so our application can access AWS services.

### Step 4.1: Create Access Key for Provider User

1. In the users list, click on **provider-test-user**
2. Click the **Security credentials** tab
3. Scroll to **Access keys** section
4. Click **Create access key** button

![](/worklogAws/images/5-Workshops/lab2/task3/step4.1.png)

---

### Step 4.2: Choose Use Case

1. Select **Application running outside AWS** (for our Java Spring Boot app)
2. Uncheck ✓ **I understand the above recommendation** (acknowledge)
3. Click **Next**


---

### Step 4.3: Create and Store Access Key

1. Click **Create access key**
2. You'll see:
   - **Access key ID**
   - **Secret access key**
3. **IMPORTANT:** Save these securely! You won't be able to see the secret again
4. Consider using AWS Secrets Manager for production


---

### Step 4.4: Create Access Key for Consumer User

1. Click on **consumer-test-user**
2. Click **Security credentials** tab
3. Click **Create access key**
4. Follow same process as Step 4.2-4.3
5. Save consumer user's access keys


---

## Summary

You have successfully created two test IAM users:

| User Name | Role/Policy | Purpose | Access Type |
|-----------|-------------|---------|------------|
| **provider-test-user** | EvDataProviderPolicy | Test data provider uploads | Console + Programmatic |
| **consumer-test-user** | EvDataConsumerPolicy | Test data consumer downloads | Console + Programmatic |

Each user has:
- AWS Management Console access (with auto-generated password)
- Access keys for CLI/API/SDK access
- Appropriate policies attached

**Important:** Store credentials securely and never expose access keys in code or version control.

**Lab 2 Complete!** Your IAM infrastructure is now ready for use with the marketplace backend.
