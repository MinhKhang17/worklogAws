## Hướng Dẫn Từng Bước: Tạo IAM Users

### Mục Tiêu
Tạo các IAM users riêng lẻ để kiểm tra marketplace với các roles khác nhau: một user data provider và một user data consumer.

---

## Bước 1: Mở IAM Users Console

1. Trong AWS IAM Console, trên menu bên trái, nhấp vào **Users**
2. Bạn sẽ thấy dashboard users
3. Nhấp vào nút **Create user**

![](/images/5-Workshops/lab2/task3/step1.png)

---

## Bước 2: Tạo Data Provider Test User

### Bước 2.1: Cấu Hình Chi Tiết User

1. Trong **User name**, nhập: `provider-test-user`
2. Đánh dấu ✓ **Provide user access to the AWS Management Console**
3. Chọn **I want to create an IAM user** (được khuyến nghị)
4. Trong **Console password**, chọn **Auto-generated**
5. Bỏ đánh dấu ✓ **Users must create a new password at next sign-in** (cho kiểm tra)
6. Nhấp vào nút **Next**

![](/images/5-Workshops/lab2/task3/step2.1.png)

---

### Bước 2.2: Gán Provider Policy

1. Bạn ở trang **Set permissions**
2. Chọn **Add user to group** (nếu có nhóm) hoặc **Attach policies directly**
3. Trong hộp tìm kiếm, tìm kiếm: `EvDataProviderRole`
4. Bạn sẽ thấy role mà chúng ta đã tạo - nhưng chờ, chúng ta cần gán role thông qua các nhóm cho users
5. Hãy sử dụng **Attach policies directly** thay vào đó
6. Tìm kiếm: `EvDataProviderPolicy`
7. Đánh dấu ✓ policy
8. Nhấp vào nút **Next**

![](/images/5-Workshops/lab2/task3/step2.2.png)

---

### Bước 2.3: Xem Lại và Tạo Provider User

1. Bạn ở trang **Review and create**
2. Xem lại tất cả cài đặt
3. Nhấp vào nút **Create user**


---

### Bước 2.4: Lưu Provider User Credentials

1. Bạn sẽ thấy trang thành công với credentials:
   - User name: `provider-test-user`
   - Console sign-in URL
   - Password (autogenerated)

2. **Quan Trọng:** Lưu các credentials này một cách an toàn (bạn có thể hiển thị credentials hoặc gửi email)
3. Ghi chú sign-in URL để kiểm tra sau


---

## Bước 3: Tạo Data Consumer Test User

### Bước 3.1: Tạo Consumer User

1. Nhấp vào nút **Create user**
2. Trong **User name**, nhập: `consumer-test-user`
3. Đánh dấu ✓ **Provide user access to the AWS Management Console**
4. Chọn **I want to create an IAM user**
5. Chọn **Auto-generated** password
6. Bỏ đánh dấu ✓ **Users must create a new password at next sign-in**
7. Nhấp vào **Next**


---

### Bước 3.2: Gán Consumer Policy

1. Trên trang **Set permissions**
2. Chọn **Attach policies directly**
3. Tìm kiếm: `EvDataConsumerPolicy`
4. Đánh dấu ✓ policy
5. Nhấp vào **Next**


---

### Bước 3.3: Tạo Consumer User

1. Xem lại cài đặt
2. Nhấp vào **Create user**


---

### Bước 3.4: Lưu Consumer Credentials

1. Lưu credentials cho `consumer-test-user`
2. Ghi chú sign-in URL


---

## Bước 4: Tạo Access Keys Cho Programmatic Access

Chúng ta cần tạo access keys cho các users này để ứng dụng của chúng ta có thể truy cập các dịch vụ AWS.

### Bước 4.1: Tạo Access Key Cho Provider User

1. Trong danh sách users, nhấp vào **provider-test-user**
2. Nhấp vào tab **Security credentials**
3. Cuộn đến phần **Access keys**
4. Nhấp vào nút **Create access key**

![](/images/5-Workshops/lab2/task3/step4.1.png)

---

### Bước 4.2: Chọn Use Case

1. Chọn **Application running outside AWS** (cho ứng dụng Java Spring Boot của chúng ta)
2. Bỏ đánh dấu ✓ **I understand the above recommendation** (xác nhận)
3. Nhấp vào **Next**


---

### Bước 4.3: Tạo và Lưu Access Key

1. Nhấp vào **Create access key**
2. Bạn sẽ thấy:
   - **Access key ID**
   - **Secret access key**
3. **QUAN TRỌNG:** Lưu các khóa này một cách an toàn! Bạn sẽ không thể nhìn thấy secret lại
4. Hãy cân nhắc sử dụng AWS Secrets Manager cho production


---

### Bước 4.4: Tạo Access Key Cho Consumer User

1. Nhấp vào **consumer-test-user**
2. Nhấp vào tab **Security credentials**
3. Nhấp vào **Create access key**
4. Làm theo quy trình tương tự như Bước 4.2-4.3
5. Lưu access keys của consumer user


---

## Tóm Tắt

Bạn đã tạo thành công hai test IAM users:

| Tên User | Role/Policy | Mục Đích | Loại Truy Cập |
|----------|-------------|---------|---------------|
| **provider-test-user** | EvDataProviderPolicy | Kiểm tra tải lên của data provider | Console + Programmatic |
| **consumer-test-user** | EvDataConsumerPolicy | Kiểm tra tải xuống của data consumer | Console + Programmatic |

Mỗi user có:
- Truy cập AWS Management Console (với auto-generated password)
- Access keys cho CLI/API/SDK access
- Các policies thích hợp được gắn

**Quan Trọng:** Lưu credentials một cách an toàn và không bao giờ để lộ access keys trong code hoặc version control.
